**Setup/teardown pattern:**
```bash
setup() {
  load 'test_helper/bats-support/load'
  load 'test_helper/bats-assert/load'
  SCRIPT="${BATS_TEST_DIRNAME}/../scripts/format.sh"
  STUB_DIR="$(mktemp -d)"
  export CLAUDE_PLUGIN_ROOT="${BATS_TEST_DIRNAME}/.."
}
teardown() {
  rm -rf "${STUB_DIR}"
}
```

**Required test cases (use PATH-stub mocking for all "present" tests):**

Non-blocking guarantee (TEST-07):
- "exits 0 when ruff absent (python)" — PATH=STUB_DIR only (empty), pipe .py JSON, assert_success
- "exits 0 when rustfmt absent (rust)" — same pattern for .rs
- "exits 0 when prettier absent (typescript)" — same for .ts
- "exits 0 when gofmt absent (go)" — same for .go
- "exits 0 when ruff crashes (python)" — create stub that exits 1, assert_success still
- "exits 0 when prettier crashes (typescript)" — create stub that exits 1, assert_success still

Per-language formatter invocation (TEST-01):
- "runs ruff format for .py file when present" — create ruff stub (exit 0, touch a marker file), assert_success, verify marker exists
- "runs rustfmt for .rs file when present" — create rustfmt stub, assert_success
- "runs prettier for .ts file when present" — create prettier stub, assert_success
- "runs prettier for .js file when present" — create prettier stub, assert_success
- "runs gofmt for .go file when present" — create gofmt stub, assert_success
- "runs prettier for .json file when present" — create prettier stub, assert_success

Silent success (FMTH-07):
- "produces no stdout on successful format" — create ruff stub (exit 0), pipe .py JSON, assert_success AND assert_output ""

Skip generated directories (FMTH-09):
- "skips node_modules path" — pipe JSON with node_modules/.../file.js path, assert_success, assert_output ""
- "skips .venv path" — same with .venv/ path
- "skips target path" — same with target/ path

**Stdin JSON format** (from ARCHITECTURE.md):
```json
{"tool_name":"Write","tool_input":{"file_path":"/tmp/test.py"}}
```

**Critical pattern:** Always use `run bash -c "printf '%s' '${json}' | bash '${SCRIPT}'"` — never bare pipes with run (RESEARCH.md Pitfall 1). Pass PATH inline: `PATH='${STUB_DIR}:${PATH}'` inside the bash -c string (RESEARCH.md Pitfall 2).

For "formatter present" tests, create a stub that touches a marker file to prove it was invoked:
```bash
printf '#!/usr/bin/env bash\ntouch "%s/ruff_called"\nexit 0\n' "${STUB_DIR}" > "${STUB_DIR}/ruff"
chmod +x "${STUB_DIR}/ruff"
```
After run, check `[ -f "${STUB_DIR}/ruff_called" ]` to verify the stub was actually invoked.
