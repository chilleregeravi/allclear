<task type="auto" tdd="true">
  <name>Task 2: Create drift-versions.sh with multi-format version extraction and comparison</name>
  <files>skills/drift/scripts/drift-versions.sh</files>
  <behavior>
    - Given two sibling repos with package.json containing the same package at different versions, drift-versions.sh reports CRITICAL drift with both versions and repo names
    - Given a sibling repo with go.mod using both inline and block require formats, all dependencies are extracted
    - Given Cargo.toml with simple string deps and inline table deps (version = "X"), both forms are extracted (yq path and grep fallback)
    - Given pyproject.toml with [project.dependencies] section, PEP 508 deps are extracted
    - Given two repos sharing a package at the same version, no drift is reported for that package
    - Given a package appearing in only one repo, it is silently skipped (not drift)
    - Given semver range specifiers (^1.2.3 vs ~1.2.3), the mismatch is reported as WARN (different locking strategy) not CRITICAL
    - With --all flag, INFO-level findings are shown
    - Without --all flag, INFO-level findings are suppressed
  </behavior>
  <action>
Create `skills/drift/scripts/drift-versions.sh` following the complete skeleton from the research (Pattern 2 and Pattern 3). The script must:

1. Source `drift-common.sh` from the same directory: `source "$(dirname "${BASH_SOURCE[0]}")/drift-common.sh"`
2. Call `parse_drift_args "$@"` to set SHOW_INFO
3. Implement `extract_versions()` function accepting a repo directory path, returning `PACKAGE_NAME=VERSION` lines for:
   - **package.json**: Use `jq -r '(.dependencies // {}) + (.devDependencies // {}) | to_entries[] | "\(.key)=\(.value)"'`
   - **go.mod**: Use awk pattern handling both inline `require` and block `require ( ... )` forms (see research Pattern 2)
   - **Cargo.toml**: Use `yq` when available; grep fallback for simple `name = "version"` and inline table `version = "X"` forms
   - **pyproject.toml**: Use `yq` when available; grep/awk fallback for PEP 508 lines in [project.dependencies]
4. Use bash associative arrays (`declare -A pkg_versions pkg_repos`) to collect all packages across all sibling repos
5. For each package appearing in 2+ repos: compare versions
   - If versions differ and both are exact pins: emit CRITICAL
   - If versions differ and one/both use range specifiers (^, ~, >=): emit WARN with note about different locking strategy
   - Strip leading `^~>=<` for comparison to avoid false positives between `^1.2.3` and `1.2.3`
6. If no drift found, print "No version drift detected across N repos."
7. Script must be executable, use `set -euo pipefail`, and route debug output to stderr only

Per research anti-patterns: do NOT report packages appearing in only one repo. Do NOT use `sort -V` for matching (use for display ordering only). DO normalize range specifiers before comparing.
  </action>
