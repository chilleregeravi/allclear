  <name>Task 1: Create drift-types.sh for heuristic type definition consistency checking</name>
  <files>skills/drift/scripts/drift-types.sh</files>
  <action>
Create `skills/drift/scripts/drift-types.sh` implementing the type definition consistency checker. The script must:

1. Source `drift-common.sh` from same directory and call `parse_drift_args "$@"`
2. Implement per-language type extraction functions:
   - `extract_ts_types()`: grep for `export (interface|type) [A-Z]` in `*.ts` files under `src/` (maxdepth 4). Output `TYPENAME` one per line
   - `extract_go_structs()`: grep for `type [A-Z]\w+ struct` in `*.go` files. Output `TYPENAME` one per line
   - `extract_py_classes()`: grep for `class [A-Z]\w+.*:` in `*.py` files under `src/` or project root (maxdepth 4). Output `TYPENAME` one per line
   - `extract_rs_structs()`: grep for `pub struct [A-Z]` in `*.rs` files under `src/`. Output `TYPENAME` one per line

3. Implement `detect_repo_language()` that checks for manifest files (package.json -> ts, go.mod -> go, pyproject.toml -> py, Cargo.toml -> rs). Returns language tag.

4. Group sibling repos by language. Only compare repos that share the same language — per research Pitfall 4, cross-language comparison generates noise.

5. For same-language repo groups: find type names appearing in 2+ repos. For each shared type name:
   - Extract the full definition body (the lines between the declaration and closing brace/dedent) from each repo
   - Compare field lists using `diff`
   - If fields differ: emit CRITICAL with the specific field differences
   - If fields match: emit INFO (only shown with --all)

6. Cap type name comparison at top 50 per language group to avoid slowness (research Pitfall 5). Use `sort | uniq -d` to find shared names.

7. Limit file scan depth to maxdepth 4 to keep execution fast.

8. If no shared types found across any language group, print "No shared type definitions found across sibling repos."

Script must be executable, use `set -euo pipefail`, route debug output to stderr.
  </action>

  </verify>
  <done>drift-types.sh extracts type/interface/struct/class names from TS, Go, Python, and Rust repos; groups repos by language; compares shared type field lists within same-language groups; reports CRITICAL for field mismatches with specific differences shown.</done>
</task>

<task type="auto">
  <name>Task 2: Create drift-openapi.sh with oasdiff and graceful fallback</name>
  <files>skills/drift/scripts/drift-openapi.sh</files>
  <action>
Create `skills/drift/scripts/drift-openapi.sh` implementing the OpenAPI spec comparison. The script must:

1. Source `drift-common.sh` from same directory and call `parse_drift_args "$@"`

2. Implement `find_openapi_spec()` accepting a repo directory, returning the path to the OpenAPI spec file. Probe these locations in order (per research Pattern 5):
   - `openapi.yaml`, `openapi.yml`, `openapi.json`
   - `swagger.yaml`, `swagger.yml`, `swagger.json`
   - `api/openapi.yaml`, `api/openapi.yml`, `api/openapi.json`
   - `api/swagger.yaml`, `docs/openapi.yaml`, `spec/openapi.yaml`
   - Fallback: `find "$repo_dir" -maxdepth 3 -name "openapi.yaml" -o -name "openapi.json" | head -1`
   - Return 1 if not found

3. Collect repos that have OpenAPI specs. If fewer than 2 repos have specs, print "Fewer than 2 repos have OpenAPI specs — nothing to compare." and exit 0.

4. Implement `compare_openapi()` accepting two spec paths and two repo names:
   - **When `oasdiff` is available** (`command -v oasdiff`):
     - Run `oasdiff breaking "$spec_a" "$spec_b"` — if output exists, emit CRITICAL with breaking changes
     - Run `oasdiff diff "$spec_a" "$spec_b" --format text | head -20` — if output exists, emit WARN with non-breaking diffs
   - **When `oasdiff` is absent but `yq` is available**:
     - Print INFO to stderr: "oasdiff not installed — using basic structural comparison"
     - Extract all YAML/JSON paths from both specs via `yq '.. | path | join(".")' "$spec"` and diff them
     - If structural differences found, emit WARN with count of differing paths and suggest installing oasdiff
   - **When neither `oasdiff` nor `yq` is available**:
     - Emit INFO: "Cannot compare OpenAPI specs without oasdiff or yq. Install either tool."

5. Compare all pairs of repos that have specs. For N repos, compare all N*(N-1)/2 pairs. If N > 5, compare each against the first repo only (hub-and-spoke) to limit execution time.

6. Per research Pitfall 3: never do raw YAML diff of OpenAPI specs — always use oasdiff for $ref resolution or clearly flag the comparison as degraded.

Script must be executable, use `set -euo pipefail`, route debug output to stderr.
  </action>
  <verify>
    <automated>bash -n skills/drift/scripts/drift-openapi.sh && grep -q "find_openapi_spec\|oasdiff" skills/drift/scripts/drift-openapi.sh && echo "OK"</automated>
  </verify>
  <done>drift-openapi.sh discovers OpenAPI specs across sibling repos, uses oasdiff for structured comparison when available, falls back to yq-based structural diff, and reports CRITICAL for breaking changes and WARN for non-breaking differences. Gracefully handles missing tools without errors.</done>
