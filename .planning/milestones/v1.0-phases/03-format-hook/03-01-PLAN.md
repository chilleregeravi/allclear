<task type="auto">
  <name>Task 2: Create format.sh auto-format hook script</name>
  <files>scripts/format.sh</files>
  <action>
Create `scripts/format.sh` as an executable bash script (`chmod +x`). This is the primary deliverable for Phase 3.

The script must implement ALL of the following in this exact order:

**1. Shebang and header:**
- `#!/usr/bin/env bash`
- Comment: AllClear auto-format hook, PostToolUse, non-blocking (always exits 0)

**2. Early exit for disable toggle (forward compatibility with Phase 8 CONF-02):**
- `[[ "${ALLCLEAR_DISABLE_FORMAT:-}" == "1" ]] && exit 0`

**3. Read stdin once into a variable:**
- `INPUT=$(cat)` — stdin is a one-time stream, must be captured in one read
- Extract file path: `FILE=$(printf '%s\n' "$INPUT" | jq -r '.tool_input.file_path // empty')`
- The `// empty` is CRITICAL — without it, jq returns the literal string "null" when the field is absent
- Guard: `[[ -z "$FILE" || ! -f "$FILE" ]] && exit 0`

**4. Path exclusion (FMTH-09):**
- Loop over skip patterns: `/node_modules/`, `/.venv/`, `/venv/`, `/env/`, `/target/`, `/.git/`, `/__pycache__/`, `/.tox/`
- Use substring match: `[[ "$FILE" == *"$SKIP_PAT"* ]] && exit 0`
- Note: `.env/` is NOT excluded because `.env` at project root is a file (the trailing slash in patterns prevents false positives on `.env` files)

**5. Extension-based formatter dispatch:**
- Extract extension: `EXT="${FILE##*.}"`
- Use a `case "$EXT" in` statement with these branches:

  - `py)` — Check `command -v ruff`, run `ruff format "$FILE" >/dev/null 2>&1 || true`. Fallback: check `command -v black`, run `black "$FILE" >/dev/null 2>&1 || true`.
  - `rs)` — Check `command -v rustfmt`, run `rustfmt "$FILE" >/dev/null 2>&1 || true`. rustfmt exits 1/2/3 on various issues — ALL must be suppressed.
  - `ts|tsx|js|jsx|mjs|cjs)` — Check `command -v prettier`, run `prettier --write "$FILE" >/dev/null 2>&1 || true`. Fallback 1: check `[[ -x "./node_modules/.bin/prettier" ]]`. Fallback 2: check `command -v eslint`, run `eslint --fix "$FILE" >/dev/null 2>&1 || true`.
  - `go)` — Check `command -v gofmt`, run `gofmt -w "$FILE" >/dev/null 2>&1 || true`. gofmt always exits 0 but guard with `|| true` for consistency.
  - `json|yaml|yml)` — Check `command -v prettier`, run `prettier --write "$FILE" >/dev/null 2>&1 || true`. Fallback: check local `./node_modules/.bin/prettier`.

**6. Non-blocking exit guarantee (FMTH-10):**
- Final line: `exit 0` — unconditional, always reached
- Do NOT use `set -e` or `set -euo pipefail` anywhere in this script — it breaks the non-blocking guarantee when formatters exit non-zero

**Critical anti-patterns to avoid:**
- Do NOT redirect only stderr (`2>/dev/null`) — must redirect BOTH stdout and stderr (`>/dev/null 2>&1`) to ensure silent operation (FMTH-07)
- Do NOT use `jq -r '.tool_input.file_path'` without `// empty` — returns literal "null" string
- Do NOT omit `|| true` on any formatter call — even gofmt which normally exits 0
- Do NOT use `set -e` — rustfmt exit codes 1/2/3 will propagate and break the hook
