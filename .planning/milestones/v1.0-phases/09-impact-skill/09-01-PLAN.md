<objective>
Create the `/allclear impact` cross-repo reference scanning skill. This skill scans all sibling repositories for references to specified symbols or auto-detected changed symbols, classifies matches by type (code, config, docs, test), and groups results by repo.

Purpose: This is AllClear's primary differentiator -- no other Claude Code plugin offers cross-repo reference scanning. It catches cross-repo breaking changes before they ship.

Output: Three files -- lib/siblings.sh (shared discovery library), scripts/impact.sh (scan engine), skills/cross-impact/SKILL.md (Claude skill definition).

<task type="auto">
  <name>Task 1: Create sibling discovery library and impact scan engine</name>
  <files>lib/siblings.sh, scripts/impact.sh</files>
  <action>
Create two bash scripts. Both must use `set -euo pipefail` and `#!/usr/bin/env bash`.

**lib/siblings.sh** -- Shared sibling repo discovery library (also used by Phase 2, 10, 11, 12):

Create a `list_siblings` function that:
1. Accepts optional `current_dir` argument (defaults to `$PWD`)
2. Computes `parent_dir=$(dirname "$current_dir")`
3. Checks for `${current_dir}/allclear.config.json` -- if present, read siblings array using `printf '%s\n' "$(cat "$config_file")" | jq -r '.siblings[]?.path // empty'` (per PLGN-07 pattern)
4. If no config, auto-discover by scanning `$parent_dir/*/` for directories containing `.git/`
5. CRITICAL: Must exclude the current repo from results: `[[ "$(cd "$d" && pwd)" != "$(cd "$current_dir" && pwd)" ]]`
6. Output: one absolute path per line, one per sibling repo
7. All debug output to stderr only (per PLGN-08)

**scripts/impact.sh** -- Core scan engine:

1. Set `PLUGIN_ROOT="${CLAUDE_PLUGIN_ROOT:-$(cd "$(dirname "${BASH_SOURCE[0]}")/.." && pwd)}"` for local dev fallback
2. Source `"${PLUGIN_ROOT}/lib/siblings.sh"`
3. Parse CLI args in a while loop:
   - `--changed` sets CHANGED=true
   - `--exclude <repo>` appends to EXCLUDES array
   - Positional args append to TERMS array
4. If `--changed` is true, extract symbols from `git diff HEAD~1`:
   - Get changed source files only (filter by extension: py, rs, ts, js, tsx, jsx, go, java, rb, sh)
   - For each changed file, extract lines starting with `+` or `-` (skip `---`/`+++` markers)
   - Grep for language-specific declaration keywords: `def `, `class `, `fn `, `func `, `pub fn `, `pub struct `, `pub enum `, `interface `, `type `, `export (const|function|class) `
   - Extract the identifier name after the keyword
   - Deduplicate with `sort -u`
   - NOTE on macOS: `mapfile` is not available in bash 3.2. Use a `while read` loop instead to populate the TERMS array from extracted symbols
5. If no terms after all parsing, print error JSON to stderr and exit 1
6. Get siblings via `list_siblings`
7. Print header: "Scanning for: ${TERMS[*]}" then "---"
8. For each sibling path:
   - Skip if repo name is in EXCLUDES array
   - Print "repo: $repo_name"
   - For each term, run `grep -rn` with:
     - `--include` for: *.py, *.rs, *.ts, *.tsx, *.js, *.jsx, *.go, *.json, *.yaml, *.yml, *.toml, *.md, *.rst
     - `--exclude-dir` for: .git, node_modules, .venv, target, dist, build, .planning
   - Pipe grep output through classification using a `classify_match` function (see below)
   - Output format per match: `{repo}\t{term}\t{type}\t{filepath}` -- deduplicated by filepath per term using `sort -u -t$'\t' -k4`
   - Print empty line after each repo
9. Create `classify_match` function (can be defined at top of script):
   - Priority order: test > docs > config > code (default)
   - Test: path contains `/test/`, `/tests/`, `/__tests__/`, `/spec/`, or filename matches `*_test.*`, `*_spec.*`, `*.test.*`, `*.spec.*`
   - Docs: extension is md, rst, txt, adoc
   - Config: extension is json, yaml, yml, toml, ini; or filename is Makefile, Dockerfile
   - Code: everything else
   - IMPORTANT on macOS: `${var,,}` lowercase syntax is bash 4+. Use `tr '[:upper:]' '[:lower:]'` or `awk 'tolower()'` instead for bash 3.2 compatibility

Make both files executable (chmod +x).

   d. Summarize by repo: which repos have matches, match types, file counts

   f. If no matches found, confirm the symbol appears safe to change
6. Output interpretation section:
   - Match types: code (direct source reference -- HIGH risk), config (configuration reference -- MEDIUM risk), test (test reference -- update needed), docs (documentation reference -- LOW risk)
   - Group by repo, then by match type within each repo
