Create `skills/quality-gate/SKILL.md` with the following structure. This is an LLM prompt playbook, not a shell script.

**Frontmatter (YAML):**
```yaml
---
name: quality-gate
description: Run quality checks for this project. Use when the user invokes /allclear, asks to verify code quality, or wants to confirm code is clean before committing.
allowed-tools: Bash
argument-hint: "[lint|format|test|typecheck|quick|fix]"
---
```

**Body sections (in order):**

1. **Project Context** — Shell injection block using `!` backtick syntax:
   - Project type: `!`source "${CLAUDE_PLUGIN_ROOT}/lib/detect.sh" && detect_project_type . 2>/dev/null || { for f in pyproject.toml Cargo.toml go.mod package.json; do [ -f "$f" ] && echo "${f%%.*}" | sed 's/pyproject/python/;s/Cargo/rust/;s/go/go/;s/package/node/'; done; }``. The fallback inline detection handles the case where Phase 2 (lib/detect.sh) has not been executed yet.
   - Makefile targets: `!`[ -f Makefile ] && make -qp 2>/dev/null | grep -E '^[a-zA-Z_-]+:[^=]' | grep -E '^(lint|format|fmt|test|typecheck|check|quick|fix):' | cut -d: -f1 | sort -u | tr '\n' ' ' || echo "none"``
   - Working directory: `!`pwd``

2. **Subcommand Dispatch** — A table mapping `$ARGUMENTS` values to check subsets:
   - (empty) → lint, format check, test, typecheck
   - lint → lint only
   - format → format check only (dry-run, no modification)
   - test → tests only
   - typecheck → type checking only
   - quick → lint + format check (no tests, no typecheck)
   - fix → lint --fix + format --write (applies changes)
   - Any unrecognized value → treat as empty (run all checks)

3. **Command Selection** — For each check category (Lint, Format Check, Format Fix, Test, Typecheck), a decision table:
   - First line: If Makefile target was listed in the injected targets above, use `make <target>`
   - Fallback per language:
     - **Lint:** Python: `ruff check .`, Rust: `cargo clippy -- -D warnings`, TS/JS: `npx eslint .`, Go: `golangci-lint run`
     - **Format Check:** Python: `ruff format --check .`, Rust: `cargo fmt --check`, TS/JS: `npx prettier --check .`, Go: `gofmt -l .` (fail if output non-empty)
     - **Format Fix:** Python: `ruff format .`, Rust: `cargo fmt`, TS/JS: `npx prettier --write .`, Go: `gofmt -w .`
     - **Lint Fix:** Python: `ruff check --fix .`, Rust: `cargo clippy --fix --allow-dirty`, TS/JS: `npx eslint --fix .`, Go: `golangci-lint run --fix`
     - **Test:** Python: `pytest` (or `python -m pytest`), Rust: `cargo test`, TS/JS: `npm test`, Go: `go test ./...`
     - **Typecheck:** Python: `mypy .` or `pyright` (check for config), Rust: `cargo check`, TS: `npx tsc --noEmit`, Go: `go vet ./...`
   - If no tool is found for a check, skip that check and note "skipped — no tool found" in results

4. **Execution Instructions** — Tell Claude to:
   - Run each applicable check one at a time using the Bash tool
   - Capture timing using `date +%s` before/after each command
   - Record exit code (0 = PASS, non-zero = FAIL)
   - Continue running remaining checks even if one fails
   - For mixed-language projects (multiple types detected), run checks for ALL detected languages

5. **Result Reporting** — Structured output format (GATE-04):
   ```
   ## AllClear Quality Gate Results

   | Check      | Status | Time   | Command                    |
   |------------|--------|--------|----------------------------|
   | lint       | PASS   | 1.2s   | make lint                  |
   | format     | FAIL   | 0.4s   | ruff format --check .      |

   **Summary: X/Y checks passed.**
   ```
   If any checks failed, add: "Run `/allclear fix` to auto-fix lint and format issues."
   If all checks passed, add: "All checks passed."

6. **Fix Subcommand Report** — Separate format for `fix` results:
   ```
   ## AllClear Fix Results

   Applied auto-fixes:
   - lint: [command] ([N] issues fixed)
   - format: [command] ([N] files reformatted)

   Re-run `/allclear` to verify fixes resolved all issues.
   Note: Test and typecheck failures require manual review — auto-fix is not applied.
   ```

7. **Critical Constraint (GATE-05)** — Explicit prohibition block:
   "IMPORTANT: The `fix` subcommand ONLY applies auto-fixes to lint and format. NEVER auto-fix test failures or typecheck errors. These require human review and may silently alter code semantics if auto-fixed. If test or typecheck failures exist alongside lint/format issues, fix only lint and format, then report the remaining failures for manual resolution."

8. **Edge Cases** — Instructions for:
   - No quality tools detected for project type: report "No quality tools detected for this project type" and exit cleanly
   - Tool not installed: skip that check, note "skipped — [tool] not found" in results, use `command -v [tool] >/dev/null 2>&1` to verify
   - Mixed-language project: run checks for all detected languages, group results by language if multiple

The file should be well-formatted Markdown (it is an LLM prompt). Use clear headers, tables, and bullet lists. Target approximately 150-250 lines.
