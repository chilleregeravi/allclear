QueryEngine constructor and public API:
```javascript
export class QueryEngine {
  constructor(db) { /* stores db reference, prepares statements */ }

  // Transitive impact: returns all services downstream of sourceServiceId
  // direction: 'downstream' (default) | 'upstream'
  // Returns: Array<{ id, name, depth, path }>
  transitiveImpact(sourceServiceId, { direction = 'downstream', maxDepth = 10 } = {})

  // Direct impact only (no recursion)
  // Returns: Array<{ id, name, protocol, method, path }>
  directImpact(sourceServiceId, direction = 'downstream')

  // Classify a set of changes against stored map
  // changes: Array<{ type: 'removed'|'changed'|'added', endpoint?, field?, fieldType? }>
  // Returns: Array<{ severity: 'CRITICAL'|'WARN'|'INFO', description, affectedServices }>
  classifyImpact(changes)

  // FTS5 keyword search across services, connections, fields
  // Returns: Array<{ kind: 'service'|'connection'|'field', id, name, snippet }>
  search(query, { limit = 20 } = {})

  // Write methods (used by scan manager in later phases):
  upsertRepo(repoData)         // Returns repo id
  upsertService(serviceData)   // Returns service id
  upsertConnection(connData)   // Returns connection id
  upsertSchema(schemaData)     // Returns schema id
  upsertField(fieldData)       // Returns field id
  updateRepoState(repoId, lastScannedCommit)
  createMapVersion(label)      // Copies DB via VACUUM INTO snapshot path; inserts map_versions row
}
```

Recursive CTE for downstream traversal (from design doc — use exactly this pattern):
```sql
WITH RECURSIVE impacted(id, depth, path) AS (
  -- Base: direct connections from source
  SELECT target_service_id AS id,
         1 AS depth,
         ',' || source_service_id || ',' || target_service_id || ',' AS path
  FROM connections
  WHERE source_service_id = ?

  UNION ALL

  -- Recursive: extend from already-visited nodes
  SELECT c.target_service_id,
         i.depth + 1,
         i.path || c.target_service_id || ','
  FROM connections c
  JOIN impacted i ON c.source_service_id = i.id
  WHERE i.path NOT LIKE '%,' || c.target_service_id || ',%'  -- cycle detection
    AND i.depth < ?                                            -- depth limit (:maxDepth)
)
SELECT DISTINCT i.id, s.name, MIN(i.depth) AS depth
FROM impacted i
JOIN services s ON s.id = i.id
GROUP BY i.id
ORDER BY depth;
```

For upstream traversal, swap source_service_id and target_service_id in the CTE.

Impact classification rules:
- CRITICAL: a connection with a specific (method, path) that existed before no longer exists in the current connections table for that service
- WARN: a field row for a connection's schema exists with the same name but a different `type` value than in the previous state
- INFO: a new field row was added to a schema that did not exist in the previous state

For classifyImpact, the "previous state" is passed in as the `changes` argument (caller provides the delta — query engine applies severity rules). The changes array items are:
```javascript
{ type: 'removed', serviceId, method, path }          // → CRITICAL
{ type: 'changed', serviceId, fieldName, oldType, newType } // → WARN
{ type: 'added', serviceId, fieldName }               // → INFO
```
