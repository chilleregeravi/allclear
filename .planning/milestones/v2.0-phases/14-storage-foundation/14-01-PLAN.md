DB path pattern:
```
~/.allclear/projects/<sha256(projectRootPath).slice(0,12)>/impact-map.db
```
Use `crypto.createHash('sha256').update(projectRoot).digest('hex').slice(0, 12)` for the hash.
`projectRoot` = `process.cwd()` unless `ALLCLEAR_DATA_DIR` env var is set.

Exports required from worker/db.js:
```javascript
// Opens (or creates) the database for the given project root.
// Runs pending migrations before returning.
// Idempotent — safe to call multiple times.
export function openDb(projectRoot?: string): Database
// Returns the already-opened database instance (throws if openDb not yet called).
export function getDb(): Database
```

Required SQLite pragmas (in this order):
```javascript
db.pragma('journal_mode = WAL');
db.pragma('foreign_keys = ON');
db.pragma('synchronous = NORMAL');
db.pragma('cache_size = -64000');   // 64 MB page cache
db.pragma('busy_timeout = 5000');   // 5s — prevents SQLITE_BUSY on concurrent reads
```

Schema tables (from design doc data model):
- repos(id, path, name, type, last_commit, scanned_at)
- services(id, repo_id FK→repos, name, root_path, language)
- connections(id, source_service_id FK→services, target_service_id FK→services, protocol, method, path, source_file, target_file)
- schemas(id, connection_id FK→connections, role, name, file)
- fields(id, schema_id FK→schemas, name, type, required BOOLEAN)
- map_versions(id, created_at, label, snapshot_path)
- repo_state(id, repo_id FK→repos UNIQUE, last_scanned_commit, last_scanned_at)

Migration tracker table (managed by db.js, not a migration file):
- schema_versions(version INTEGER PRIMARY KEY, applied_at TEXT)

FTS5 virtual tables (content tables — auto-sync via triggers):
```sql
CREATE VIRTUAL TABLE IF NOT EXISTS connections_fts USING fts5(
  path, protocol, source_file, target_file,
  content='connections', content_rowid='id'
);

CREATE VIRTUAL TABLE IF NOT EXISTS services_fts USING fts5(
  name,
  content='services', content_rowid='id'
);

CREATE VIRTUAL TABLE IF NOT EXISTS fields_fts USING fts5(
  name, type,
  content='fields', content_rowid='id'
);
```

FTS5 triggers to keep content tables in sync (for each FTS table — after INSERT, after DELETE, after UPDATE):
```sql
-- Example for services_fts (repeat pattern for connections_fts and fields_fts):
CREATE TRIGGER IF NOT EXISTS services_ai AFTER INSERT ON services BEGIN
  INSERT INTO services_fts(rowid, name) VALUES (new.id, new.name);
END;
CREATE TRIGGER IF NOT EXISTS services_ad AFTER DELETE ON services BEGIN
  INSERT INTO services_fts(services_fts, rowid, name) VALUES ('delete', old.id, old.name);
END;
CREATE TRIGGER IF NOT EXISTS services_au AFTER UPDATE ON services BEGIN
  INSERT INTO services_fts(services_fts, rowid, name) VALUES ('delete', old.id, old.name);
  INSERT INTO services_fts(rowid, name) VALUES (new.id, new.name);
END;
```
