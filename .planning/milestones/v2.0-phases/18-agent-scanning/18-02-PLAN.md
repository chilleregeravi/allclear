`getChangedFiles` implementation details:
- Run `git -C {repoPath} diff --name-status {sinceCommit} HEAD` via execSync
- Parse output: lines starting with 'D\t' → deleted, 'R...\t' → renamed (extract old and new path), all others → modified
- When sinceCommit is null, run `git -C {repoPath} ls-files` and return all as modified (full scan)
- Wrap in try/catch; if execSync throws (not a git repo), return `{ error: 'not a git repo' }`

`buildScanContext` implementation details:
- If options.full === true → return `{ mode: 'full', files: null }`
- Call `queryEngine.getRepoState(repoId)` — if null (no entry) → return `{ mode: 'full', files: null }` (SCAN-06: first scan is always full)
- Get current HEAD: `git -C {repoPath} rev-parse HEAD`
- If HEAD === repoState.last_scanned_commit → return `{ mode: 'skip', files: null }` (SCAN-05: unchanged repos produce zero findings)
- Otherwise → call getChangedFiles(repoPath, repoState.last_scanned_commit) and return `{ mode: 'incremental', files: result }`

```javascript
/**
 * Scan one or more repos by dispatching agents sequentially in the foreground.
 * Background agents cannot access MCP tools (Claude Code issue #13254) — foreground only.
 *
 * @param {string[]} repoPaths - absolute paths to repos to scan
 * @param {{ full?: boolean }} options
 * @param {object} queryEngine - Phase 14 query engine (getRepoState, setRepoState, upsertRepo)
 * @returns {Promise<ScanResult[]>}
 */
async function scanRepos(repoPaths, options = {}, queryEngine) { ... }
```

Implementation:
1. For each repoPath (sequentially — `for...of`, not `Promise.all`):
   a. Call `queryEngine.upsertRepo(repoPath, path.basename(repoPath), 'single')` to ensure repo row exists
   b. Call `buildScanContext(repoPath, repo.id, queryEngine, options)`
   c. If `ctx.mode === 'skip'`: push `{ repoPath, mode: 'skip', findings: null }` and continue
   d. Load and interpolate `worker/agent-prompt.md`: replace `{{REPO_PATH}}` with repoPath and `{{SERVICE_HINT}}` with basename
   e. Invoke the agent using Claude's `Task` tool (the standard way Claude Code skills spawn subagents): the agent receives the interpolated prompt and the repoPath as context. Per SCAN-08: no language restrictions in the prompt.
   f. Receive the raw agent text response
   g. Call `parseAgentOutput(rawAgentResponse)` from findings-schema.js
   h. If `result.valid === false`: push `{ repoPath, mode: ctx.mode, findings: null, error: result.error }` and continue (do not crash — one bad agent result should not stop other repos)
   i. If `result.valid === true`: call `queryEngine.setRepoState(repo.id, getCurrentHead(repoPath))` to update the last scanned commit. Push `{ repoPath, mode: ctx.mode, findings: result.findings }`
2. Return the array of ScanResult objects

Helper `getCurrentHead(repoPath)`:
```javascript
function getCurrentHead(repoPath) {
  return execSync(`git -C ${JSON.stringify(repoPath)} rev-parse HEAD`, { encoding: 'utf8' }).trim();
}
```

The agent invocation step (step e) must use the Claude Code `Task` tool pattern — write the agent invocation as a comment/stub that calls the agent with the prompt, since the actual Task tool call happens inside Claude's execution context, not in Node.js.
