<!-- MCP SDK pattern from research. Executor uses these directly. -->

From @modelcontextprotocol/sdk (v1.27.1):
```javascript
import { McpServer } from '@modelcontextprotocol/sdk/server/mcp.js';
import { StdioServerTransport } from '@modelcontextprotocol/sdk/server/stdio.js';

const server = new McpServer({ name: 'allclear-impact', version: '2.0.0' });
// tools registered here (plan 02 adds them)
const transport = new StdioServerTransport();
await server.connect(transport);
```

From better-sqlite3 (v12.8.0):
```javascript
import Database from 'better-sqlite3';
// Read-only open for MCP server (does not need to write)
const db = new Database(dbPath, { readonly: true });
db.pragma('journal_mode = WAL');
```

DB path resolution (from design doc):
```javascript
const dbPath = process.env.ALLCLEAR_DB_PATH
  || path.join(process.cwd(), '.allclear', 'impact-map.db');
```

.mcp.json registration (from architecture research):
```json
{
  "mcpServers": {
    "allclear-impact": {
      "type": "stdio",
      "command": "node",
      "args": ["${CLAUDE_PLUGIN_ROOT}/worker/mcp-server.js"],
      "env": {
        "ALLCLEAR_DB_PATH": ".allclear/impact-map.db"
      }
    }
  }
}
```
Note: ALLCLEAR_DB_PATH in .mcp.json is relative to the user's project CWD (where Claude Code runs), not the plugin root. This is intentional — the DB lives in the project, not the plugin cache.

  <name>Task 2: Create worker/mcp-server.js skeleton and .mcp.json registration</name>
  <files>worker/mcp-server.js, .mcp.json</files>
  <action>
Create `worker/mcp-server.js`. This is the standalone stdio MCP server entry point. Requirements:

1. Import `McpServer` from `@modelcontextprotocol/sdk/server/mcp.js` and `StdioServerTransport` from `@modelcontextprotocol/sdk/server/stdio.js`.
2. Import `Database` from `better-sqlite3` and `path`, `fs` from Node.js built-ins.
3. Resolve DB path: `const dbPath = process.env.ALLCLEAR_DB_PATH || path.join(process.cwd(), '.allclear', 'impact-map.db');`
4. Create a `openDb()` helper that:
   - Returns `null` if the file does not exist at dbPath (use `fs.existsSync`)
   - Opens read-only: `new Database(dbPath, { readonly: true })`
   - Sets `db.pragma('journal_mode = WAL')` (safe to set on read-only opens)
   - Returns the db instance
   - Wraps in try/catch — on any error, logs to stderr and returns null
5. Create `McpServer` instance: `new McpServer({ name: 'allclear-impact', version: '2.0.0' })`
6. At this point, register zero tools (Plan 02 adds them). Just create the server.
7. Connect `StdioServerTransport`: `await server.connect(new StdioServerTransport())`
8. Add a SIGTERM handler: `process.on('SIGTERM', () => process.exit(0))`
9. Export `openDb` as a named export so Plan 02 can import it.

CRITICAL: Zero `console.log()` calls anywhere in this file. All debug/info output goes to `console.error()`. The MCP stdio transport owns stdout entirely — any non-MCP byte on stdout corrupts the session.

Create `.mcp.json` at the project root (next to package.json, not inside .claude-plugin/). Contents:

```json
  }
}
```
Note: `ALLCLEAR_DB_PATH` here is intentionally relative (no leading slash). Claude Code resolves it relative to the project CWD, which is where the user's `.allclear/impact-map.db` lives.
