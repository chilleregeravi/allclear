**scripts/lint.sh addition:**

Add a new check block (after existing checks) that:
1. Looks for `console.log` in `worker/mcp-server.js` if that file exists.
2. If found: print an error message like "ERROR: console.log found in worker/mcp-server.js — MCP stdout pollution risk. Use console.error() instead." and exit with code 1.
3. If not found: print "OK: no console.log in worker/mcp-server.js" and continue (do not exit).
4. If `worker/mcp-server.js` does not exist yet: skip silently (do not fail — lint runs in CI before the file may be present).

Use grep pattern: `grep -n "console\.log" worker/mcp-server.js`

**tests/mcp-server.bats:**

Create the file at `tests/mcp-server.bats`. Load bats-support and bats-assert. Change to project root with `cd "$BATS_TEST_DIRNAME/.."`.

Write these tests:

1. `@test "lint.sh passes when mcp-server.js has no console.log"` — run `bash scripts/lint.sh` and assert_success (the check should pass since Plan 02 added no console.log).

2. `@test "mcp-server.js contains all 5 required tool names"` — read the file and assert each of: impact_query, impact_changed, impact_graph, impact_search, impact_scan appears.

3. `@test "mcp-server starts and responds to initialize"` — pipe the MCP initialize JSON-RPC message to the process with a 3-second timeout, assert output contains "protocolVersion".

4. `@test "mcp-server tools/list returns all 5 tools"` — pipe initialize + tools/list JSON-RPC messages (see interfaces block), assert output contains "impact_query" and "impact_search".

5. `@test "mcp-server returns empty results when DB absent"` — set `ALLCLEAR_DB_PATH=".allclear/nonexistent-test.db"`, send an impact_query call request, assert output contains `"results"` and does NOT contain `"isError"`.

MCP tool call request format:
```json
{"jsonrpc":"2.0","id":3,"method":"tools/call","params":{"name":"impact_query","arguments":{"service":"test-service","transitive":false}}}
```

NOTE: The initialize handshake must come before tools/list or tools/call in the same piped session.
