  <behavior>
    - GET /api/readiness always returns 200 {status: 'ok'} — even when queryEngine is null/undefined (no DB yet)
    - GET /graph calls queryEngine.getGraph() and returns result; if queryEngine absent returns 503 {error: 'No map data yet'}
    - GET /impact?change=foo calls queryEngine.getImpact('foo'); missing change param returns 400 {error: 'change param required'}
    - GET /service/my-service calls queryEngine.getService('my-service'); service not found returns 404 {error: 'Service not found'}
    - POST /scan returns 202 {status: 'started', message: 'Scan triggered'}
    - GET /versions calls queryEngine.getVersions(); returns array
    - Server binds to 127.0.0.1 (never 0.0.0.0)
    - Static files served from worker/ui/ at root path /
    - createHttpServer(queryEngine, options) is the exported factory function
  </behavior>
  <action>
Create `worker/http-server.js` as an ES module. Export a single async factory function `createHttpServer(queryEngine, options = {})` that:

1. Creates a Fastify instance with `{ logger: false }` (no stdout logging — logs go to stderr only via a custom logger if needed).

2. Registers `@fastify/cors` for localhost dev.

3. Registers `@fastify/static` pointing at `path.join(import.meta.dirname, 'ui')` with `{ root: ..., prefix: '/' }`.

4. Registers routes in this EXACT ORDER (order matters for readiness probe):
   a. `GET /api/readiness` — MUST be first. Returns `{ status: 'ok' }` always. Never touches queryEngine.
   b. `GET /graph` — calls `queryEngine.getGraph()`, returns result. If queryEngine is null, returns 503 `{ error: 'No map data yet' }`.
   c. `GET /impact` — reads `request.query.change`. If missing, 400. Calls `queryEngine.getImpact(change)`.
   d. `GET /service/:name` — calls `queryEngine.getService(params.name)`. If null result, 404.
   e. `POST /scan` — returns 202 `{ status: 'started', message: 'Scan triggered' }`. Actual scan logic comes in Phase 18.
   f. `GET /versions` — calls `queryEngine.getVersions()`, returns array.

5. All route handlers wrap queryEngine calls in try/catch. Errors return 500 `{ error: err.message }`.

6. The function calls `await fastify.listen({ port: options.port || 37888, host: '127.0.0.1' })` and returns the fastify instance.

7. Exports: `export { createHttpServer }`.

For `worker/http-server.test.js`, write tests using Node's built-in `test` module (no external test framework — project uses bats for shell but Node built-in test for JS unit tests). Use `import { createHttpServer } from './http-server.js'`. Create a mock queryEngine:


Inject mock into `createHttpServer(mockQE, { port: 0 })`. Use `fastify.inject()` for all test HTTP calls (no real TCP socket needed).

Test all behaviors listed in the `<behavior>` block.
