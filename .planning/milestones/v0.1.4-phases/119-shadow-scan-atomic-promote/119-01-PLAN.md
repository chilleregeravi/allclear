2. **Apply hook:** Phase 117 adds `applyPendingOverrides(scanVersionId, queryEngine)` invoked between `persistFindings` and `endScan` in `manager.js` (per the v0.1.4 ROADMAP pre-flight note for Phase 117). <background>
Phase 119 ships the operator-facing shadow workflow. Plan 119-01 owns the WRITE side (scan into shadow); Plan 119-02 owns the READ + PROMOTE side (`/diff --shadow`, `/promote-shadow`).

Per RESEARCH ¬ß1: shadow QE is **always-fresh, never cached** ‚Äî chosen over the cache-key-extension alternative because (a) it sidesteps the `openDb` singleton problem entirely, and (b) it eliminates the eviction-on-promote landmine. Caching offers near-zero benefit since shadow ops are short-lived.

Per RESEARCH ¬ß2: the scan path injection seam IS the `queryEngine` argument to `scanRepos`. No fork of `manager.js`, no flag inside `scanRepos`. The shadow vs live decision is made one level UP in the HTTP route handler.

Per RESEARCH ¬ß6 Q3: `/arcanon:shadow-scan` mirrors `/arcanon:map`'s flag set (`--full`, repo paths) ‚Äî symmetry is operator-friendly. Flags forward through unchanged via `scanRepos(options, ...)`.

Per RESEARCH ¬ß6 Q4: when shadow DB already exists, the command prints a one-line warning and proceeds (non-interactive). The previous shadow scan is overwritten in place.

       ```javascript
       /**
        * Open a fresh QueryEngine pointed at the project's SHADOW database
        * (impact-map-shadow.db). NEVER enters the pool cache ‚Äî every call
        * opens a new handle. Caller MUST close via qe._db.close() when done.
        *
        * Used by /arcanon:shadow-scan, /arcanon:diff --shadow, and tests.
        * Bypasses openDb() (which is a process-singleton keyed by module-level
        * state) by replicating the inline pragma + runMigrations pattern from
        * getQueryEngineByHash. This guarantees shadow and live can never collide.
        *
        * @param {string} projectRoot
        * @param {{ create?: boolean }} [opts] - if create=true, mkdir+open even when shadow DB absent.
        * @returns {QueryEngine|null} null if shadow DB doesn't exist and create=false.
        */
       export function getShadowQueryEngine(projectRoot, opts = {}) {
         if (!projectRoot) return null;
         const dir = projectHashDir(projectRoot);
         const shadowPath = path.join(dir, "impact-map-shadow.db");
         if (!fs.existsSync(shadowPath) && !opts.create) return null;
         try {
           fs.mkdirSync(dir, { recursive: true });
           const db = new Database(shadowPath);
           db.pragma("journal_mode = WAL");
           db.pragma("foreign_keys = ON");
           db.pragma("synchronous = NORMAL");
           db.pragma("cache_size = -64000");
           db.pragma("busy_timeout = 5000");
           runMigrations(db);
           // Note: NOT cached. Caller is responsible for closing.
           return new QueryEngine(db, null);
         } catch (err) {
           process.stderr.write(
             `[db-pool] Failed to open shadow DB for ${projectRoot}: ${err.message}\n`,
           );
           return null;
         }
       }
       ```
