<background>
INT-10 is a one-test plan owned at the bats E2E layer. Two real OpenAPI fixtures exercise the new explicit-spec entry point:

- **spec-a (baseline)**: `openapi: 3.0.0`, one path `/users` (GET) returning `{id: integer, name: string}`.
- **spec-b (drifted)**: same as spec-a but the field `name` is renamed to `full_name` — a backward-incompatible client break.

The test asserts that running `--spec specA --spec specB` exits 0 AND prints output that mentions the drift. Exact wording varies between oasdiff (rich diff output) and the yq fallback (structural mismatch line); both code paths in `drift-openapi.sh` emit at least one line containing "openapi" + a CRITICAL/WARN level + the words "drift" or "differ" or "incompatible". The test uses a tolerant `[[ "$output" == *openapi* && "$output" == *(drift|differ|incompatible|rename|break)* ]]` style match.

Test fixtures live under `plugins/arcanon/tests/fixtures/externals/` (per the task brief). The bats test file lives at `tests/drift-openapi-explicit-spec.bats` per the repo-root convention.
</background>

# tests/drift-openapi-explicit-spec.bats — Phase 121 / INT-10.
#
# Happy-path coverage for /arcanon:drift openapi --spec PATH_A --spec PATH_B.
# Implementation lives in Phase 120 (drift-openapi.sh --spec flag).
# This test asserts the explicit-spec entry point works end-to-end with two

SPEC_B="${REPO_ROOT}/plugins/arcanon/tests/fixtures/externals/openapi-spec-b.yaml"

  # (Test 4) actually exercises the "no linked repos / no specs found" branch.

@test "/arcanon:drift openapi --spec X --spec Y exits 0 with two real specs" {
  [ -f "$SPEC_A" ] || skip "fixture spec-a missing"
  [ -f "$SPEC_B" ] || skip "fixture spec-b missing"
  run bash "$DRIFT_SH" openapi --spec "$SPEC_A" --spec "$SPEC_B"
  [ "$status" -eq 0 ]
}

@test "/arcanon:drift openapi --spec X --spec Y reports the User.name -> full_name drift" {
  [ -f "$SPEC_A" ] || skip "fixture spec-a missing"
  [ -f "$SPEC_B" ] || skip "fixture spec-b missing"
  run bash "$DRIFT_SH" openapi --spec "$SPEC_A" --spec "$SPEC_B"
  [ "$status" -eq 0 ]
  # Tolerant substring match — wording differs between oasdiff (when installed)

    return 1
