<objective>
Land the `hub.evidence_mode` config flag end-to-end with a v1.2 payload version that gates the new shape, while preserving byte-identical output for every existing caller. Three modes ship together because they share the same wiring boundary in `buildFindingsBlock`:

- **`"full"` (default)** — current behavior. Evidence is a string when present, omitted when absent. Version stays at "1.0" or "1.1" per the existing libraryDepsEnabled state machine. **Byte-identical to today's output** for every existing input.
- **`"hash-only"`** — evidence string is replaced by `{hash: sha256(evidence), start_line, end_line}` on every connection that has evidence. Version bumps to "1.2".
- **`"none"`** — evidence field is omitted entirely from every connection. Version bumps to "1.2".

Two prerequisites must land in the same plan because they're load-bearing: `loadLatestFindings()` must be extended to SELECT `c.evidence`/`c.confidence`/`c.source_file` (today the column is never read so the flag would be a no-op), and a new pure helper `extractEvidenceLocation()` must be carved out of `computeVerdict()` so payload-build and verify share one algorithm for line-number derivation (the agent does not persist line numbers — they're computed at read time from the source file + evidence substring per Phase 109 §D-03).

Purpose: ship the *capability* for hash-only evidence uploads (privacy / compliance) without disturbing any existing user. Phase 121 owns hub-side adoption coordination; this plan ships the plugin emitter.

    Step 2 — Create `evidence-location.js`:
    ```javascript
    /**
     * worker/hub-sync/evidence-location.js — Compute hash + line range for an
     * evidence snippet relative to its source file. Pure function; no DB,
     * no network, no mutable state.
     *
     * Used by:
     *   - worker/hub-sync/payload.js (hash-only evidence_mode)
     *   - worker/server/http.js computeVerdict (verify command)
     *
     * Single source of truth for evidence line semantics. The agent does not
     * persist line numbers — they are derived at read time from the source
     * file + the literal evidence substring (Phase 109 §D-03).
     */

    import crypto from "node:crypto";
    import fs from "node:fs";
    import path from "node:path";

    /**
     * @param {string|null|undefined} evidence — agent-emitted snippet (may be empty)
     * @param {string|null|undefined} sourceFile — relative path recorded on the connection
     * @param {string|null|undefined} projectRoot — absolute path to resolve sourceFile against
     * @returns {{
     *   hash: string|null,
     *   start_line: number|null,
     *   end_line: number|null,
     *   evidence_present: boolean
     * }}
     */
    export function extractEvidenceLocation(evidence, sourceFile, projectRoot) {
      const trimmed = typeof evidence === "string" ? evidence.trim() : "";
      if (!trimmed) {
        return { hash: null, start_line: null, end_line: null, evidence_present: false };
      }

      // Hash is computed even when we can't locate lines — the hub can correlate
      // by hash across re-scans even without line numbers.
      const hash = crypto.createHash("sha256").update(trimmed).digest("hex");

      if (!sourceFile || !projectRoot) {
        return { hash, start_line: null, end_line: null, evidence_present: false };
      }

      const absPath = path.resolve(projectRoot, sourceFile);
      let content;
      try {
        content = fs.readFileSync(absPath, "utf8");
      } catch {
        return { hash, start_line: null, end_line: null, evidence_present: false };
      }

      const matchIdx = content.indexOf(trimmed);
      if (matchIdx === -1) {
        return { hash, start_line: null, end_line: null, evidence_present: false };
      }

      // 1-indexed line numbers (mirrors http.js:102-104 algorithm exactly).
      const start_line = (content.slice(0, matchIdx).match(/\n/g) || []).length + 1;
      const newlinesInSnippet = (trimmed.match(/\n/g) || []).length;
      const end_line = start_line + newlinesInSnippet;

      return { hash, start_line, end_line, evidence_present: true };
    }
    ```
