# Plan 118-01: `/arcanon:correct` — Stage Scan Overrides

**Phase:** 118 · **Plan:** 01 · **Owned REQs:** CORRECT-02 (subset for stage), CORRECT-06 (insert/select tests), CORRECT-07 (correct bats happy paths) · **Risk:** Medium · **Wave:** 3

## Goal

Operators can stage corrections to a scan's findings via `/arcanon:correct`, which inserts a single row into the `scan_overrides` table per invocation. The override row is consumed on the next scan by Phase 117's `applyPendingOverrides` hook (Phase 117 owns the apply path; this plan owns only the stage path).

## Success Criteria (what must be TRUE after this plan ships)

1. `/arcanon:correct connection --action delete --connection 5` inserts a single row into `scan_overrides` with `kind='connection'`, `target_id=5`, `action='delete'`, `payload=NULL` (or `'{}'`), and exits 0.
2. `/arcanon:correct connection --action update --connection 5 --source <svc> --target <svc>` inserts a row with `action='update'` and `payload='{"source":"<svc>","target":"<svc>"}'`.
3. `/arcanon:correct service --action rename --service <name> --new-name <name>` inserts a row with `kind='service'`, `target_id=<resolved-services.id>`, `action='rename'`, `payload='{"new_name":"<name>"}'`.
4. `/arcanon:correct service --action set-base-path --service <name> --base-path <path>` inserts a row with `action='set-base-path'` and `payload='{"base_path":"<path>"}'`.
5. Invalid kind (`/arcanon:correct foo ...`), unknown action, or missing required flags exit 2 with a friendly usage message naming the offending field.
6. Non-existent target (`--connection 999999` when no such connection exists, or `--service unknown-name`) exits 2 with a friendly error listing valid targets.
7. Service-name lookup that returns more than one match exits 2 with a friendly disambiguation message listing all matches by `(service_id, repo_path)`.
8. `/arcanon:correct --json ...` emits a single JSON object `{ok: true, override_id: <int>, kind, target_id, action, payload}` instead of the human line.
9. The command is silent in non-Arcanon directories (no `impact-map.db`) and exits 0 — same contract as `/arcanon:list` and `/arcanon:doctor`.
10. `commands/correct.md` exists with frontmatter (`description`, `argument-hint`, `allowed-tools: Bash`) and a body that sources `lib/worker-client.sh`, asserts project dir, and calls `bash hub.sh correct $ARGUMENTS`.
11. Bats test asserts the row goes IN (CORRECT-07 happy paths × 4 actions). Does NOT assert the row gets APPLIED — that's Phase 117's test.

## Tasks (executable, ordered)

### Task 1 — Add `cmdCorrect` to `worker/cli/hub.js`

**File:** `plugins/arcanon/worker/cli/hub.js` (modify)

**1a. Extend `main()` to pass `positional` to handlers.**

Current `main()` at `hub.js:1233-1249`:

```javascript
async function main() {
  const { sub, flags } = parseArgs(process.argv.slice(2));
  const handler = HANDLERS[sub];
  // ...
  await handler(flags);
}
```

Change to:

```javascript
async function main() {
  const { sub, flags, positional } = parseArgs(process.argv.slice(2));
  const handler = HANDLERS[sub];
  // ...
  await handler(flags, positional);
}
```

JavaScript silently ignores extra args, so existing handlers (`cmdVerify(flags)` etc.) continue to work unchanged. Only `cmdCorrect` and `cmdRescan` (Plan 02) consume `positional`.

**1b. Add the `cmdCorrect` handler.** Insert before the `HANDLERS` declaration (~line 1220). The function signature: `async function cmdCorrect(flags, positional)`.

Function body responsibilities, in order:

1. **Project detection** (silent no-op):
   ```javascript
   const cwd = process.cwd();
   const dbPath = path.join(projectHashDir(cwd), "impact-map.db");
   if (!fs.existsSync(dbPath)) process.exit(0);
   ```

2. **Validate `kind`** from `positional[0]`. Must be one of `'connection' | 'service'`. Anything else → print usage to stderr, `process.exit(2)`.

3. **Validate `--action`** flag value. Must be one of `'delete' | 'update' | 'rename' | 'set-base-path'`. Cross-validate against kind:
   - `kind='connection'` allows `delete | update`
   - `kind='service'` allows `rename | set-base-path`
   - Mismatch → friendly error like `error: action 'rename' is only valid for kind 'service' (you passed kind 'connection')`, exit 2.

4. **Resolve `target_id`** depending on kind:
   - **`kind='connection'`:** require `flags.connection` to be a positive integer (reuse `cmdVerify` pattern at `hub.js:414-419`). Then `qe._db.prepare("SELECT id FROM connections WHERE id = ?").get(id)` — if absent, exit 2 with `error: connection ID <id> not found`.
   - **`kind='service'`:** require `flags.service` to be a non-empty string (the service NAME per REQUIREMENTS CORRECT-02). Resolve via `SELECT id, name FROM services WHERE name = ?` — if zero results, exit 2 with `error: service '<name>' not found`. If >1 results, exit 2 with the disambiguation message listing each `id` + `repo_path`.

5. **Construct `payload`** per action:
   - `delete` → `null`
   - `update` → require `flags.source` AND `flags.target` (both non-empty strings); `JSON.stringify({source: flags.source, target: flags.target})`
   - `rename` → require `flags["new-name"]`; `JSON.stringify({new_name: flags["new-name"]})`
   - `set-base-path` → require `flags["base-path"]`; `JSON.stringify({base_path: flags["base-path"]})`
   - Missing required fields → exit 2 with `error: --action <X> requires --<field>` listing every missing field.

6. **Insert via `qe.upsertOverride`** (Phase 117's API per assumption P117-B):
   ```javascript
   const { getQueryEngine } = await import("../db/pool.js");
   const qe = getQueryEngine(cwd);
   const overrideId = qe.upsertOverride({
     kind, target_id: targetId, action, payload, created_by: "cli",
   });
   ```

7. **Emit result** via `emit(json, flags, human)`:
   - JSON shape: `{ok: true, override_id: overrideId, kind, target_id: targetId, action, payload}`
   - Human shape: `correct: queued (override_id=<id>) — kind=<kind>, target_id=<id>, action=<action>` followed by a hint line like `Apply on next /arcanon:map or /arcanon:rescan run.`

8. **Exit 0** on success.

**1c. Register in HANDLERS:**

```javascript
const HANDLERS = {
  // ... existing entries ...
  list: cmdList,     // NAV-01
  doctor: cmdDoctor, // NAV-03
  correct: cmdCorrect, // CORRECT-02 stage path (Phase 118-01)
};
```

**Done when:** `node plugins/arcanon/worker/cli/hub.js correct connection --action delete --connection 1` (against a seeded DB) prints the human line and exits 0; `node ... correct foo` exits 2 with usage error; `node ... correct service --action rename --service notfound --new-name X` exits 2 with the not-found error.
