Update the `/graph` route handler in `worker/server/http.js` to read boundaries from `allclear.config.json` and merge into the response.

In the `/graph` handler (around line 93), after `const graph = qe.getGraph();`:

```javascript
// Read boundaries from project config
let boundaries = [];
try {
  const projectRoot = request.query?.project || request.query?.hash || '';
  // Resolve project root — for hash-based lookups, get the project path from QE
  let configDir = projectRoot;
  if (qe && typeof qe.getProjectRoot === 'function') {
    configDir = qe.getProjectRoot();
  }
  if (configDir) {
    const cfgPath = path.join(configDir, 'allclear.config.json');
    const cfg = JSON.parse(fs.readFileSync(cfgPath, 'utf8'));
    boundaries = Array.isArray(cfg.boundaries) ? cfg.boundaries : [];
  }
} catch { /* no config file or no boundaries key — return empty array */ }
return reply.send({ ...graph, boundaries });
```

Key details:
- `fs` and `path` are already imported at top of http.js.
- Always return `boundaries: []` when no config or no boundaries key (Research Pitfall 6).
- Wrap in try/catch so missing config file does not crash the route.
- The `...graph` spread preserves existing `services`, `connections`, `repos`, `mismatches` fields.

If `qe.getProjectRoot()` does not exist, fall back to using `request.query.project` directly (it is the absolute path when provided). For hash-based lookups, check if QueryEngine exposes project root. If not, boundaries will be empty for hash-based lookups — acceptable for now.
