# Milestone v0.1.1: Command Cleanup + Update + Ambient Hooks

**Status:** ✅ SHIPPED 2026-04-21
**Phases:** 97-100
**Total Plans:** 12

## Overview

Tighten the Arcanon command surface, add a self-update flow, and make
cross-repo impact awareness ambient during implementation. The user no
longer needs to remember `/arcanon:cross-impact` — its features fold
into `/arcanon:impact`, and impacted files now surface a warning
automatically when Claude edits them.

## Milestone Summary

**Key Decisions:**

- **Merge cross-impact into impact (not just delete)** — External 20-point review flagged that the cross-impact command had features (`--exclude`, `--changed`, 3-state degradation) that regressed without absorption. A serialization guard ensures 97-01 deletion runs AFTER 97-04 merge inside the same wave. Rationale: zero feature regression across the rename.
- **Deprecated stub for `/arcanon:upload` (not hard-remove)** — CI pipelines with hard-coded `/arcanon:upload` would break. Stub with stderr deprecation warning forwards to `/arcanon:sync` for one release.
- **`auto_upload` → `auto_sync` with two-read fallback** — `cfg?.hub?.["auto-sync"] ?? cfg?.hub?.["auto-upload"]` with stderr warning on legacy key read. Prevents silent breakage for users with old config.
- **Tier 1 pure-bash, Tier 2 SQLite, Tier 3 worker HTTP** — Keep the hot path out of Node.js to hit the 50ms latency target. SQLite fallback when worker is down.
- **PreToolUse (not UserPromptSubmit)** — Fires only on actual Edit/Write tool invocations, not every prompt. Avoids "Arcanon: nothing impacted" noise.
- **macOS 130ms p99 caveat accepted** — Pure-bash fork overhead is platform-dependent. Linux CI should hit 50ms; macOS is a documented deviation not a regression.

**Issues Resolved:**

- Session-start banner read stale JSON key (`hub_auto_upload` instead of `hub_auto_sync`) — caught by cross-phase integration audit, fixed in commit `802d59b` before milestone close.
- `date +%s%3N` returns literal garbage on macOS BSD — added `_ms_now()` helper with validation and python3 fallback in impact-hook.sh.
- 97-02 bats Task 3 would fail in Wave 1 because the full suite asserted cross-impact was deleted (happens Wave 2). Narrowed verify to file-existence + @test count; full bats deferred to 97-01 Wave 2 verification.
