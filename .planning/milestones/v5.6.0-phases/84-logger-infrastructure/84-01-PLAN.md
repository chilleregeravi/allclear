<objective>
Extend `plugins/ligamen/worker/lib/logger.js` with:

1. **LOG-01** — Size-based rotation: before each write, if `worker.log` is >= 10 MB rename the chain of rotated files (`.3` deleted, `.2` → `.3`, `.1` → `.2`, active → `.1`) then write to a fresh `worker.log`.
2. **LOG-02** — TTY-aware stderr: only call `process.stderr.write()` when `process.stderr.isTTY` is truthy, eliminating the double-write when the worker runs under nohup.

Purpose: Prevents unbounded log growth and fixes the duplicate-line bug introduced by `worker-start.sh` redirecting stderr into the same log file.
Output: Updated `logger.js` + updated `tests/worker/logger.test.js` (path fix + new tests for both behaviours).
</objective>
