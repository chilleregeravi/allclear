<objective>
Apply the three-step HiDPI canvas fix (MDN pattern) to make graph nodes and edges pixel-crisp on Retina/HiDPI displays, and increase baseline font sizes for readability.

Purpose: The current `resize()` in graph.js sets `canvas.width = container.clientWidth` which maps 1 canvas pixel to 1 CSS pixel. On a Retina display (devicePixelRatio=2), the browser stretches those logical pixels to fill the physical pixel grid, producing blurry rendering. This is a visual regression on any MacBook. The fix is 5 lines in graph.js and 2 lines in renderer.js.

Output: renderer.js and graph.js updated so the graph is crisp on all displays. No changes to coordinate math — state.transform, state.positions, and mouse coordinates remain in CSS pixel space.
</objective>

  <action>
Update the `resize()` function inside `init()` in worker/ui/graph.js. Replace the current two-line assignment with the three-step HiDPI pattern:

```javascript
function resize() {
  const dpr = window.devicePixelRatio || 1;
  const cssW = container.clientWidth;
  const cssH = container.clientHeight;
  canvas.width = Math.round(cssW * dpr);
  canvas.height = Math.round(cssH * dpr);
  canvas.style.width = cssW + 'px';
  canvas.style.height = cssH + 'px';
  render();
}
```

After the `window.addEventListener("resize", resize)` line, add the DPR-change watcher so the graph re-renders when the user moves the window between monitors:

```javascript
function watchDPR() {
  const mq = window.matchMedia(`(resolution: ${window.devicePixelRatio}dppx)`);
  mq.addEventListener('change', () => { resize(); watchDPR(); }, { once: true });
}
watchDPR();
```
