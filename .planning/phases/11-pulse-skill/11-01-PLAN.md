  <action>
Create `scripts/pulse-check.sh` as a sourceable shell library (not an executable entry point) that provides the following functions. The script must be POSIX-compatible bash with `set -euo pipefail` at the top.

**Function: `pulse_check_kubectl`**
- Check `command -v kubectl > /dev/null 2>&1`
- If not found, print: "kubectl not found in PATH. Install kubectl to use /allclear pulse." and "See: https://kubernetes.io/docs/tasks/tools/"
- Return 1 if not found, 0 if found

**Function: `pulse_resolve_namespace`**
- Accept one argument: the environment name (may be empty)
- If argument provided, echo it as the namespace
- If empty, detect from kubectl context: `kubectl config view --minify -o jsonpath='{.contexts[0].context.namespace}' 2>/dev/null`
- Default to "default" if detection returns empty
- Echo the resolved namespace

**Function: `pulse_list_deployments`**
- Accept one argument: namespace
- Run: `kubectl get deployments -n "$1" -o jsonpath='{range .items[*]}{.metadata.name}{"\n"}{end}' 2>/dev/null`
- Echo deployment names, one per line

**Function: `pulse_get_image_tag`**
- Accept two arguments: deployment name, namespace
- Extract image: `kubectl get deployment "$1" -n "$2" -o jsonpath='{.spec.template.spec.containers[0].image}' 2>/dev/null`
- Extract tag with `"${IMAGE##*:}"` — if image has no colon (no tag), echo "latest"
- Echo the tag

**Function: `pulse_get_latest_git_tag`**
- Try `git describe --tags --abbrev=0 2>/dev/null` first
- If empty, fall back to `git tag --sort=version:refname 2>/dev/null | tail -1`
- If still empty, echo "no-tags"
- Echo the tag

**Function: `pulse_check_health`**
- Accept three arguments: deployment name, namespace, pod name
- Start `kubectl port-forward -n "$NS" "pod/$POD" 18080:8080 > /dev/null 2>&1 &` and capture PID
- Sleep 1 second for port-forward to establish
- Try health endpoints in order: /health, /healthz, /actuator/health, /ready
- For each, use `curl -s -o /tmp/allclear_health_body -w "%{http_code}" --max-time 5 "http://localhost:18080${PATH_ATTEMPT}" 2>/dev/null`
- On HTTP 200, read body and break
- Kill port-forward PID with `kill "$PF_PID" 2>/dev/null` (always, even on failure)
- Parse health body with jq (PLGN-07 pattern): `printf '%s\n' "$BODY" | jq -r '.status // empty' 2>/dev/null`
- Normalize status: up/ok/healthy/pass -> "HEALTHY", down/error/fail* -> "UNHEALTHY", empty + HTTP 200 -> "HEALTHY", empty + non-200 -> "UNHEALTHY", other -> "UNKNOWN"
- Extract components if present: `printf '%s\n' "$BODY" | jq -r '.components // empty' 2>/dev/null`
- Echo a structured line: "STATUS|endpoint_path|raw_status|components_json_or_empty"

**Function: `pulse_get_pod_for_deployment`**
- Accept two arguments: deployment name, namespace
- Run: `kubectl get pods -n "$2" -l "app=$1" -o jsonpath='{.items[0].metadata.name}' 2>/dev/null`
- If empty, try without label selector: `kubectl get pods -n "$2" -o jsonpath='{range .items[*]}{.metadata.name}{"\n"}{end}' 2>/dev/null | grep "$1" | head -1`
- Echo pod name or empty string

All functions must route debug output to stderr only (per PLGN-08). No function should call `exit` — use `return` codes so the sourcing script controls flow.

Use a header comment block: `# AllClear Pulse Skill — service health check helper` with `# Sourced by SKILL.md — not intended for direct execution`.

Create `skills/pulse/SKILL.md` following the Claude Code skill format. This is the orchestration prompt that tells Claude how to run service health checks.

**Frontmatter (YAML between --- delimiters):**
```yaml
---
name: pulse
description: This skill should be used when the user asks to "check service health", "check if services are running", "run pulse", "check cluster health", "are my services healthy", or asks about the status of running Kubernetes services.
disable-model-invocation: true
allowed-tools: Bash
argument-hint: "[environment] [service-name]"
---
```

`disable-model-invocation: true` because this is a user-invoked action with cluster side effects — must not auto-fire during code editing.

**Body structure (imperative third-person, ~1500-2000 words):**

**Section 1: Overview**
Brief description: "Checks the health and version status of Kubernetes services. Queries pod health endpoints and compares running image tags to the latest git tag."

**Section 2: Prerequisites**
Instruct Claude to source the helper script:
```
source "${CLAUDE_PLUGIN_ROOT}/scripts/pulse-check.sh"
```
Then call `pulse_check_kubectl`. If it returns non-zero, display the skip message and stop. Do not proceed with any further steps.

**Section 3: Resolve Target**
- Parse `$ARGUMENTS`: if two words, first is environment/namespace, second is service name. If one word, determine if it looks like an environment (dev/staging/prod/default) or a service name. If empty, check all deployments in default namespace.
- Call `pulse_resolve_namespace` with the environment argument.
- Store namespace in `NS` variable.

**Section 4: Discover Services**
- If a specific service name was provided, use only that deployment.
- Otherwise call `pulse_list_deployments "$NS"` to get all deployments.
- If no deployments found, report "No deployments found in namespace $NS" and stop.

**Section 5: For Each Service — Health Check**
For each deployment:
1. Call `pulse_get_pod_for_deployment "$DEPLOY" "$NS"` to find a running pod
2. If no pod found, report "No running pods for $DEPLOY" and continue to next
3. Call `pulse_check_health "$DEPLOY" "$NS" "$POD"` to get health status
4. Parse the structured output line

**Section 6: For Each Service — Version Comparison**
For each deployment:
1. Call `pulse_get_image_tag "$DEPLOY" "$NS"` to get running tag
2. Call `pulse_get_latest_git_tag` to get latest local tag
3. Compare: if equal, "UP TO DATE"; if different, "DRIFT: running $RUNNING vs latest $LATEST"
4. Handle edge cases: "no-tags" means no git tags found — report "cannot compare, no git tags"
5. Handle digest-based images (contains sha256) — report "uses digest, cannot compare to semver tag"

**Section 7: Output Format**
Instruct Claude to present results as a formatted table:

```
## Pulse: Service Health Report

Namespace: {NS}

| Service | Health | Endpoint | Version | Status |
|---------|--------|----------|---------|--------|
| api     | HEALTHY | /health | v1.2.3  | UP TO DATE |
| worker  | UNHEALTHY | /healthz | v1.2.0 | DRIFT (latest: v1.2.3) |
| frontend | - | no pod | v1.2.3 | UP TO DATE |
```

If components were found in health response, list them below the table under a "Component Details" subsection.

**Section 8: Error Handling**
- kubectl command failures: report which service failed, continue to next
- Port-forward timeout: report "health check timed out", continue
- No deployments at all: single-line message, not a table
- All healthy: include a summary line "All N services healthy and up to date"

**Writing guidelines:**
- Use imperative form ("Check kubectl availability", "Extract the image tag")
- Keep instructions step-by-step so Claude follows them linearly
- Do NOT include raw bash code blocks longer than 3 lines — reference the pulse-check.sh functions instead
- Target 1500-2000 words total
