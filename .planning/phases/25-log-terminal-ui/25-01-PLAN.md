  <name>Task 3: Create log-terminal.js module</name>
  <files>worker/ui/modules/log-terminal.js</files>
  <action>
    Create `worker/ui/modules/log-terminal.js`. This module exports one function: `initLogTerminal()`.

    Full implementation requirements:

    **Constants:**
    ```javascript
    const MAX_LOG_LINES = 500;
    const POLL_INTERVAL_MS = 2000;
    ```

    **initLogTerminal():**
    1. Grab DOM refs: `#log-panel`, `#log-panel-header`, `#log-lines`, `#log-component-filter`, `#log-search`, `#log-clear`, `#log-chevron`.
    2. Wire panel toggle: clicking `#log-panel-header` toggles `state.logPanelOpen`. When open: add class `log-panel--open`, remove `log-panel--collapsed`; when closed: vice versa. Start polling when opened; stop polling when closed (use a single `setInterval` handle stored in a local variable, cleared on close). Stop propagation on `.log-panel__controls` click so controls don't toggle the panel.
    3. Wire `#log-component-filter` change: set `state.logComponentFilter`; call `applyFilters()`.
    4. Wire `#log-search` input (debounced 200ms): set `state.logSearchFilter`; call `applyFilters()`.
    5. Wire `#log-clear` click: clear `#log-lines` innerHTML, reset `state.logLastSince = null`.

    **poll()** (called every `POLL_INTERVAL_MS` when panel is open):
    - Build URL: `/api/logs` with `?limit=200`.
    - If `state.logLastSince` is non-null, append `&since=${encodeURIComponent(state.logLastSince)}`.
    - If `state.logComponentFilter !== "all"`, append `&component=${state.logComponentFilter}`.
    - `fetch(url)` — on network error: silently skip (do not throw, log terminal should not crash the app).
    - Parse `data.lines` — for each line object `{ ts, level, component, msg }`:
      - Call `appendLogLine(line)`.
      - Track the maximum `ts` value seen. After processing all lines, update `state.logLastSince` to that max ts.

    **appendLogLine(line):**
    - Create `<div class="log-line">`.
    - Format text as: `[ts] [component] msg` where ts is the last 12 chars of the ISO string (time portion only).
    - Assign level class: `level === "error"` → add `log-line--error`; `level === "warn"` → add `log-line--warn`; else add `log-line--info`.
    - Store `line.component` on the element as `el.dataset.component`.
    - Store `line.msg` on the element as `el.dataset.msg` (for search filtering).
    - Call `applyFiltersToLine(el)` to apply current filters before appending.
    - Append to `#log-lines`.
    - Ring buffer: `while (logContainer.children.length > MAX_LOG_LINES) { logContainer.removeChild(logContainer.firstChild); }`.
    - Auto-scroll: call `maybeScrollToBottom()`.

    **applyFiltersToLine(el):**
    - Component filter: if `state.logComponentFilter !== "all"` and `el.dataset.component !== state.logComponentFilter`, add class `log-line--hidden`.
    - Search filter: if `state.logSearchFilter` is non-empty and `el.dataset.msg` does not include `state.logSearchFilter` (case-insensitive), add class `log-line--hidden`.
    - Otherwise remove `log-line--hidden`.

    **applyFilters():**
    - Iterate all children of `#log-lines` and call `applyFiltersToLine(el)` on each.

    **maybeScrollToBottom():**
    - `isAtBottom`: `logContainer.scrollHeight - logContainer.scrollTop - logContainer.clientHeight < 24`.
    - If `isAtBottom`, set `logContainer.scrollTop = logContainer.scrollHeight`.

    **Critical implementation notes:**
    - The `setInterval` handle must be stored in a closure variable (not state). Start it when panel opens; clear it when panel closes. Do not start a second interval if already running.
    - The polling fetch is HTTP, NOT SSE/EventSource. No zombie connection risk for polling.
    - Do NOT import `render` or touch the canvas from this module. Log terminal is DOM-only.
    - No `console.log` or `console.error` in this module (errors silently swallowed in poll()).
    - The `{ passive: false }` flag is NOT needed here — this panel does not call `e.preventDefault()` on scroll events.
  </action>
