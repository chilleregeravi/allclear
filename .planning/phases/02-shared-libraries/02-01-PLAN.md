<task type="auto">
  <name>Task 1: Create lib/detect.sh with language and project type detection</name>
  <files>lib/detect.sh</files>
  <action>
Create `lib/detect.sh` as a sourceable bash library (not an executable script). Include the shebang `#!/usr/bin/env bash` for editor syntax support but add a source guard: `[[ "${BASH_SOURCE[0]}" != "${0}" ]] || { echo "Source this file; do not execute directly." >&2; exit 1; }`.

Implement three public functions:

1. `detect_language FILE` — Maps file extension to lowercase language token using bash case statement on `"${file##*.}"`. Tokens: py->python, rs->rust, ts/tsx->typescript, js/jsx/mjs/cjs->javascript, go->go, json->json, yaml/yml->yaml, *->unknown.

2. `detect_project_type DIR` — Checks manifest files in priority order (python > rust > node > go) and returns the first match. Defaults DIR to "." if not provided. Returns "unknown" if no manifests found. Manifests: pyproject.toml or setup.py for python, Cargo.toml for rust, package.json for node, go.mod for go.

3. `detect_all_project_types DIR` — Checks ALL manifests (no early return) and emits space-separated tokens. Returns empty string if none found (not "unknown"). Uses bash array `types=()` with `types+=("token")` and outputs `"${types[*]}"`.

CRITICAL constraints per research:
- NO `set -e`, `set -u`, or `set -euo pipefail` at file level (Pitfall 1 — leaks into sourcing context)
- NO echo/printf to stdout except return values (Pitfall 2 — stdout pollution)
- All diagnostic output to stderr via `>&2` (PLGN-08)
- No hardcoded absolute paths (sourcing script owns path resolution)
- Keep file under 60 lines
  </action>
  <verify>
    <automated>bash -c 'source lib/detect.sh && [[ "$(detect_language "test.py")" == "python" ]] && [[ "$(detect_language "app.tsx")" == "typescript" ]] && [[ "$(detect_language "foo.unknown")" == "unknown" ]] && echo "detect_language: PASS"' && bash -c 'cd /tmp && mkdir -p allclear_test && touch allclear_test/pyproject.toml allclear_test/package.json && source /Users/ravichillerega/sources/allclear/lib/detect.sh && [[ "$(detect_project_type /tmp/allclear_test)" == "python" ]] && [[ "$(detect_all_project_types /tmp/allclear_test)" == "python node" ]] && rm -rf /tmp/allclear_test && echo "detect_project_type: PASS"'</automated>
  </verify>
  <done>lib/detect.sh exists with three exported functions. detect_language maps all specified extensions correctly. detect_project_type returns correct primary type with priority ordering. detect_all_project_types returns all matching types space-separated. No set -e, no stdout pollution, source guard present.</done>
</task>

<task type="auto">
  <name>Task 2: Create lib/siblings.sh with sibling repo discovery</name>
  <files>lib/siblings.sh</files>
  <action>
Create `lib/siblings.sh` as a sourceable bash library. Include shebang and source guard identical to detect.sh.

Implement one public function:

`discover_siblings [PROJECT_DIR]` — Scans the parent directory of PROJECT_DIR (defaults to CWD) for directories containing `.git/`. Returns newline-separated absolute paths, excluding the current project. Caps at max_siblings=10 to prevent SessionStart timeout (Pitfall 4).

Implementation details:
- Resolve PROJECT_DIR to absolute path using POSIX-safe `$(cd "$dir" && pwd)` fallback instead of `realpath` (Open Question 2 from research — realpath not guaranteed on macOS without Homebrew)
- Compute parent_dir via `dirname` of the resolved project path
- Glob `"$parent_dir"/*/` and check each for `.git/` directory
- Exclude current project by comparing resolved paths
- Use `(( count++ )) || true` to prevent set -e in caller from treating 0-increment as failure (bash quirk documented in research)
- Keep file under 40 lines

Same constraints as Task 1: no set -e at file level, no stdout except return values, stderr for diagnostics.
  </action>

  </verify>
  <done>lib/siblings.sh exists with discover_siblings function. Function returns newline-separated absolute paths of sibling git repos. Current project excluded. Max 10 siblings cap enforced. Uses POSIX-safe path resolution (no realpath dependency). No set -e, no stdout pollution, source guard present.</done>
