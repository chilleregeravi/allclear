<!-- From cross-impact-v2.md Map Flow section and configuration section. -->

Config file: allclear.config.json (project root, committed to git)
```json
{
  "linked-repos": ["../api", "../auth", "../sdk"],
  "impact-map": {
    "history": true
  }
}
```

Discovery sources (in priority order):
1. `allclear.config.json` linked-repos (resolved to absolute paths)
2. Claude memory context (MCP memory tool if available — read from process context, NOT a runtime tool call)
3. Parent directory scan: list subdirs of `path.join(process.cwd(), '..')`, filter for those containing a `package.json`, `pyproject.toml`, `go.mod`, `Cargo.toml`, or `pom.xml`, exclude current project dir

Repo representation:
```javascript
{
  path: "/absolute/path/to/repo",  // always absolute
  name: "repo-name",               // basename of path
  source: "config" | "discovered", // origin — used to flag new repos
  isNew: boolean                   // true if not already in config
}
```

Key design decisions (from STATE.md locked decisions):
- User confirms ALL findings before persistence — hard gate, not toggle
- Config is project-level (allclear.config.json), settings are machine-level (~/.allclear/settings.json)
- `--view` flag opens graph UI without triggering discovery/scan/confirmation
</interfaces>

  <name>Task 1: Implement repo discovery module with unit tests</name>
  <files>worker/repo-discovery.js, worker/repo-discovery.test.js</files>
  <behavior>
    - loadFromConfig(projectRoot): reads allclear.config.json, returns [] when file missing or linked-repos absent; returns array of absolute-path repo objects with source: "config", isNew: false
    - discoverNew(projectRoot, existingPaths): scans parent directory (path.join(projectRoot, '..')) for subdirs containing a project manifest file (package.json, pyproject.toml, go.mod, Cargo.toml, pom.xml); excludes projectRoot itself; excludes paths already in existingPaths (normalized to absolute); returns repos with source: "discovered", isNew: true
    - deduplicateRepos(repos): normalizes all paths (path.resolve), deduplicates by path, preserves config repos over discovered for same path
    - saveConfirmed(projectRoot, confirmedPaths): writes confirmedPaths array (strings) to allclear.config.json under linked-repos; merges with existing config (preserves impact-map and other keys); creates file if absent
    - isViewOnlyMode(args): returns true if args array contains "--view"; false otherwise
    - Test: loadFromConfig returns [] when allclear.config.json missing
    - Test: loadFromConfig returns resolved absolute paths from config
    - Test: discoverNew excludes current project dir and existing config paths
    - Test: discoverNew returns only dirs containing a manifest file
    - Test: deduplicateRepos prefers config source over discovered for same path
    - Test: saveConfirmed round-trip: write then read preserves linked-repos and other keys
    - Test: isViewOnlyMode returns true only when --view is present
  </behavior>
  <action>
    Create `worker/repo-discovery.js` as a CommonJS module (require/module.exports — matches existing worker conventions, no ESM in Node.js worker files per project pattern).

    Implement these exports:
    - `loadFromConfig(projectRoot)` — reads `path.join(projectRoot, 'allclear.config.json')` with fs.readFileSync; catches ENOENT and returns []; resolves each path in linked-repos relative to projectRoot using path.resolve(projectRoot, p); returns array of {path, name: path.basename(resolved), source: 'config', isNew: false}
    - `discoverNew(projectRoot, existingPaths)` — gets entries of path.join(projectRoot, '..') with fs.readdirSync({withFileTypes: true}); filters for directories; resolves each to absolute; skips projectRoot itself (path.resolve comparison); skips paths in existingPaths (use a Set of path.resolve values); for each remaining dir, checks existence of ANY of ['package.json', 'pyproject.toml', 'go.mod', 'Cargo.toml', 'pom.xml'] via fs.existsSync; returns qualifying entries as {path, name: path.basename, source: 'discovered', isNew: true}
    - `deduplicateRepos(repos)` — builds a Map keyed by path.resolve(r.path); for each repo, if key already exists, keep whichever has source === 'config' (config wins); returns Array.from(map.values())
    - `saveConfirmed(projectRoot, confirmedPaths)` — reads existing config (or {}); sets config['linked-repos'] = confirmedPaths (strings, not repo objects); writes JSON.stringify with 2-space indent to allclear.config.json
    - `isViewOnlyMode(args)` — returns args.includes('--view')

    Create `worker/repo-discovery.test.js` using Node.js built-in `node:test` and `assert` (no external test framework — matches project test pattern from v1.0 bats/shell tests; for Node.js unit tests use node:test). Use `fs.mkdtempSync` + cleanup in each test for temp dir fixtures.
