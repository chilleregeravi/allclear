Make these changes to `worker/ui/modules/detail-panel.js`:

**1. Add escapeHtml helper** at the top of the file (after imports, before showDetailPanel):
```javascript
function escapeHtml(str) {
  if (!str) return '';
  return String(str).replace(/&/g, '&amp;').replace(/</g, '&lt;').replace(/>/g, '&gt;');
}
```
This addresses the XSS concern documented in STATE.md. Apply to ALL new `${}` insertions in the new renderers.

**2. Replace the isLib routing block** (current lines 43-49):
Delete:
```javascript
const isLib = nodeType === "library" || nodeType === "sdk";
if (isLib) {
  html += renderLibraryConnections(outgoing, incoming, nameById);
} else {
  html += renderServiceConnections(outgoing, incoming, nameById);
}
```
Replace with:
```javascript
if (nodeType === 'infra') {
  html += renderInfraConnections(node, outgoing, nameById);
} else if (nodeType === 'library' || nodeType === 'sdk') {
  html += renderLibraryConnections(node, outgoing, incoming, nameById);
} else {
  html += renderServiceConnections(outgoing, incoming, nameById);
}
```

**3. Update the "No connections" guard** (current lines 51-55):
Change from:
```javascript
if (outgoing.length === 0 && incoming.length === 0) {
```
To:
```javascript
if (outgoing.length === 0 && incoming.length === 0 && (node.exposes || []).length === 0) {
```
This prevents "No connections" from showing after a "Manages" or "Exports" section on nodes that have exposes but no edge connections.

**4. Rewrite renderLibraryConnections()** — change signature from `(outgoing, incoming, nameById)` to `(node, outgoing, incoming, nameById)`:

The function should render:
- **Exports section** (from `node.exposes` filtered to `kind === 'export'`):
  - Split into functions (path contains `(`) and types (path does not contain `(`)
  - Show "Exports (N)" header, then "Functions (N)" sub-header with items, then "Types (N)" sub-header with items
  - Use `escapeHtml(ex.path)` for each item in a `conn-path` div
  - Only show Exports section if there are export items
- **Used by section** (from `incoming` edges, existing dedup logic with Set):
  - Keep the existing dedup-by-name pattern using `const users = new Set()`
  - Use `escapeHtml(source)` for service names
  - Use `escapeHtml(e.source_file)` for file paths
- Remove the old "Provides" section that rendered outgoing edges — library nodes should not show outgoing edges as "Provides" anymore (they show actual exports from node.exposes instead)

**5. Add renderInfraConnections(node, outgoing, nameById)** — new function after renderLibraryConnections:

The function should render:
- **Manages section** (from `node.exposes` filtered to `kind === 'resource'`):
  - Group by prefix using `r.path.split('/')[0]` (e.g., "k8s:deployment", "tf:output")
  - Show "Manages (N)" header, then for each prefix group: "prefix (N)" sub-header with items
  - Use `escapeHtml(r.path)` for each item in a `conn-path` div
  - Only show Manages section if there are resource items
- **Wires section** (from `outgoing` edges):
  - Show "Wires (N)" header with each edge showing method, path, target, and source_file
  - Use `escapeHtml()` on method, path, target, and source_file values
  - Only show Wires section if there are outgoing edges

Use the exact HTML class names from existing renderers: `detail-section`, `detail-label`, `connection-item`, `conn-method`, `conn-path`, `conn-direction`, `conn-target`, `conn-file`.

**DO NOT modify renderServiceConnections()** — the service panel must remain unchanged.
