  <name>Task 1: Add impact-map auto-start to session-start.sh</name>
  <files>scripts/session-start.sh</files>
  <behavior>
    - When allclear.config.json exists AND has .["impact-map"] key AND worker is not already running: fire worker_start_background() and continue (non-blocking)
    - When allclear.config.json has no impact-map section: skip worker start entirely (no log, no error)
    - When worker is already running: skip worker start silently (worker-start.sh handles dedup via PID, but avoid redundant call from hook)
    - Hook still exits 0 always — any worker start failure is swallowed (non-blocking trap remains)
    - Worker status line added to additionalContext when worker is running: "AllClear worker: running (port PORT, last scan: AGO)" using worker_running() and a simple worker_status() call; omit if worker not running
    - ALLCLEAR_DISABLE_SESSION_START still suppresses everything including worker start
  </behavior>
  <action>
    After the existing deduplication block (FLAG_FILE check), and BEFORE the project detection block, add:

    1. Source lib/worker-client.sh using the same CLAUDE_PLUGIN_ROOT-relative pattern already used for lib/detect.sh:
       ```bash
       WORKER_CLIENT_LIB=""
       if [[ -n "${CLAUDE_PLUGIN_ROOT:-}" ]] && [[ -f "${CLAUDE_PLUGIN_ROOT}/lib/worker-client.sh" ]]; then
         WORKER_CLIENT_LIB="${CLAUDE_PLUGIN_ROOT}/lib/worker-client.sh"
       else
         WORKER_CLIENT="${SCRIPT_DIR}/../lib/worker-client.sh"
         [[ -f "$WORKER_CLIENT" ]] && WORKER_CLIENT_LIB="$WORKER_CLIENT"
       fi
       ```

    2. Add conditional worker start — runs only if lib is found AND config has impact-map section:
       ```bash
       WORKER_STATUS=""
       if [[ -n "$WORKER_CLIENT_LIB" ]]; then
         source "$WORKER_CLIENT_LIB"
         CONFIG_FILE="${CWD}/allclear.config.json"
         if [[ -f "$CONFIG_FILE" ]] && jq -e '.["impact-map"]' "$CONFIG_FILE" >/dev/null 2>&1; then
           if ! worker_running 2>/dev/null; then
             worker_start_background 2>/dev/null || true
           else
             WORKER_STATUS=$(worker_status_line 2>/dev/null || echo "")
           fi
         fi
       fi
       ```

    3. Append WORKER_STATUS to CONTEXT string if non-empty:
       ```bash
       [[ -n "$WORKER_STATUS" ]] && CONTEXT="${CONTEXT} ${WORKER_STATUS}"
       ```

    Key constraints:
    - All worker calls wrapped in `2>/dev/null || true` — never let worker errors surface to hook stdout
    - lib/worker-client.sh absence is safe (WORKER_CLIENT_LIB="" branch skips silently)
    - Use CWD (already resolved earlier in the script) to locate allclear.config.json, not $PWD
    - Per INTG-02: the impact-map section is written by /allclear:map (Phase 20); session-start.sh only READS it. No writing here.
